## 1. 핵심 기능 (Core Features)

### 🔍 디버깅 및 트레이싱 (Tracing)
* **가시성 확보:** LLM 체인이 실행되는 모든 단계를 시각적으로 보여줍니다. 어떤 프롬프트가 들어갔고, 어떤 도구(Tool)를 사용했으며, 최종 결과가 어떻게 나왔는지 한눈에 파악할 수 있습니다.
* **오류 추적:** 중간 단계에서 실패가 발생했을 때, 정확히 어느 지점(예: 임베딩, 검색, LLM 호출 등)에서 문제가 생겼는지 즉시 확인할 수 있습니다.

### 📊 평가 (Evaluation)
* **데이터셋 관리:** 실제 사용자 로그나 테스트 케이스를 기반으로 데이터셋을 구축합니다.
* **자동 평가:** LLM-as-a-judge(LLM을 판사로 활용) 방식을 통해 답변의 정확성, 유해성, 관련성 등을 자동으로 점수화합니다.
* **회귀 테스트:** 코드를 수정했을 때 성능이 이전보다 나빠지지 않았는지 비교 분석할 수 있습니다.

### 📝 프롬프트 관리 (Prompt Hub)
* 코드를 수정하지 않고도 UI 상에서 프롬프트를 작성, 테스트, 버전 관리할 수 있습니다. 
* 팀원들과 프롬프트를 공유하고 최적의 프롬프트를 실험하는 공간으로 활용됩니다.

### 📈 모니터링 및 인사이트
* **비용 및 성능 추적:** 모델별 토큰 사용량, 비용, 지연 시간(Latency) 등을 실시간 대시보드로 제공합니다.
* **사용자 피드백:** 실제 사용자가 남긴 "좋아요/싫어요" 데이터를 트레이스(Trace)와 연결해 성능 개선의 근거로 삼습니다.

## 2. 2026년 기준 주요 업데이트 및 트렌드
현재 LangSmith는 단순한 모니터링 툴을 넘어 **에이전트(AI Agents) 운영**에 특화된 기능을 강화하고 있습니다.

* **에이전트 가시성:** 복잡한 멀티 에이전트 워크플로우에서 각 에이전트가 내린 결정 과정을 심층 분석합니다.
* **온라인 평가 (Online Evals):** 배포된 실제 환경에서 실시간으로 성능을 측정하고 이상 징후를 감지합니다.
* **강화된 보안:** 엔터프라이즈 환경을 위한 데이터 격리 및 하이브리드 배포(SaaS + Self-hosted) 옵션이 더욱 정교해졌습니다.
